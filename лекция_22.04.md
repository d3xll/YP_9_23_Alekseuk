# Инкапсуляция C#
**Инкапсуляция** - скрытие внутреннего состояния и функций объекта и предоставление доступа только к через открытый набор функций.

В C# для инкапсуляции используют ```public``` свойства и методы объекта. А переменные остаются изолированными от пользователя: ```private```

В зависимости от того какой модификатор доступа мы используем для переменной, свойства или метода мы можем сденлать их публичными, закрытыми или доступными только в производных классах и т.д.

**Инкапсуляция в ООП позволяет скрыть** от конечного пользователя реализацию той или иной части объекта (класса), обеспечивания тем самым целостность работы.

**Инкапсуляция в C# реализуется** с использованием различных модификаторов доступа к полям и свойства объкта.


###### Пример класса с инкапсуляцией:
```
public class BankAccount
{
    private double balance; // Приватное поле, не доступно извне

    public BankAccount(double initialBalance)
    {
        balance = initialBalance;
    }

    // Метод для добавления средств на счет
    public void Deposit(double amount)
    {
        if (amount > 0)
        {
            balance += amount;
        }
    }

    // Метод для получения текущего баланса
    public double GetBalance()
    {
        return balance;
    }
}
```

# Модификаторы C#
## Модификатор private
```private``` - является уровнем доступа с минимальными правами.
Доступ к закрытым членам получить только внутри тела класса или структуры, в которой они объявлены.
###### Пример:
```
class Employee
{
    private int _i;
    double _d;   // По умолочанию C# использует модификатор private, если его не объявлять
}
```
Вложенные типы в том же теле могут также обращаться к таким закрытым членам.

Ошибка времени компиляции возникнет в том случае, если создать ссылку на закрытый член за пределами класса или структуры, в которой он объявлен.

## Модификатор protected
```protected```- является модификатором доступа к члену.

Доступ к защищенному элементу может быть получен из соответствующего класса, а также экземплярами производных классов.

## Модификатор internal
```internal``` является модификатором доступа для типов и членов типов.

Внутренние типы или члены доступны только в файлах в той же сборке
